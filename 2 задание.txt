DNS (Domain Name System) - технология, которая позволяет цифровому IP-адресу устройства, на котором расположен тот или иной сайт, присвоить буквенное имя (домен).

Разница протоколов HTTP/HTTPS

HTTP (от англ. HyperText Transfer Protocol — протокол передачи гипертекста) ― это протокол передачи данных в интернете. С его помощью браузер получает информацию от сервера и показывает пользователю запрос. Это первый протокол, который создали для работы в веб-пространстве.

HTTPS (от англ. HyperText Transfer Protocol Secure — безопасный протокол передачи гипертекста) ― это тот же протокол HTTP, но защищенный SSL-сертификатом (цифровой документ, который обеспечивает безопасное соединение и обмен данными между сайтом и пользователями путём шифрования передаваемой информации).

Протокол HTTP от HTTPS отличается тем, что:

-протокол HTTP не обеспечивает защиту передаваемых данных. Любая личная информация может быть перехвачена злоумышленниками;
-HTTP работает по порту 80, а HTTPS ― по порту 443.

Версии протокола HTTP:
1) Стандарт HTTP/0.9 – версия протокола HTTP 0.9 была разработана в 1991 году. Протокол блы создан для облегчения доступа и создания навигации при помощи гипертекста. Протокол версии 0.9 был призван упорядочить взаимодействие между клиентом и сервером в сети.
2) В 1996 году был выпущен информационный документ RFC 1945 (стандарт HTTP/1.0). Данный документ стал основой для реализации приложений и компонентов с использованием протокола HTTP версии 1.0. 
3) В 1997 году была выпущена версия протокола HTTP1: был разработан стандарт HTTP/1.1.Основным нововведением в версию протокола HTTP 1.1 был режим постоянного соединения (можете почитать про постоянные HTTP соединения), другими словами: за одно соединение можно было отправлять несколько HTTP запросов и получать несколько HTTP ответов в том порядке, в котором делались запросы. 
4) 2015 году была опубликована финальная версия черновика протокола HTTP 2.
